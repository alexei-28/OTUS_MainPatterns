## Детальная архитектура микросервисов 

### User Service
**Endpoints:**
- POST /users/register – регистрация
- POST /users/login – авторизация
- GET /users/{id} – профиль
> События в Message Broker: UserRegistered, UserUpdated
>
>Потенциальные узкие места: рост числа пользователей → использовать шардирование БД и кэширование (Redis).
>
>OCP: добавление новых типов аутентификации без изменения основного сервиса.

### Tournament Service
**Endpoints:** 
- GET /tournaments – список турниров
- POST /tournaments – создать турнир
- POST /tournaments/{id}/join – подать заявку
- GET /tournaments/{id}/results – результаты
>События: TournamentCreated, TournamentStarted, TournamentFinished
>
>Потенциальные узкие места: массовые заявки → CQRS + очередь сообщений для обработки.
> 
>OCP: новые типы турниров через стратегию расчёта рейтинга.

### Battle Service
**Endpoints:**
- POST /battles/start
- GET /battles/{id}/status
- GET /battles/{id}/replay
>События: BattleFinished
> 
>Узкие места: высокое потребление CPU при симуляции боев → масштабирование через Kubernetes/Containerization.
>
>OCP: добавление новых типов боев через отдельные стратегии боев.

### Notification Service
**Endpoints:**
- нет, работает через события
>События: подписка на UserRegistered, Tournament, BattleFinished
>
>OCP: добавление новых каналов уведомлений (SMS, push) без изменения основной логики.

### Rating Service
**Endpoints:**
- GET /ratings/player/{id}
- GET /ratings/tournament/{id}
>Логика: подписка на события завершения боев и турниров.
>
>OCP: новые алгоритмы расчета рейтинга через стратегический паттерн.

### Текстовое пояснение
- Общая концепция: архитектура разделена на микросервисы по бизнес-доменам: пользователи, турниры, бои, уведомления и рейтинги.
- Коммуникация: микросервисы взаимодействуют через REST и асинхронные события через Message Broker.
- Масштабируемость: критические по нагрузке сервисы (Battle Service, Tournament Service) легко масштабируются горизонтально.
- Гибкость (Opened-Closed Principle): все сервисы имеют возможность добавления новых типов турниров, боев, уведомлений и алгоритмов рейтинга без изменения существующего кода.
- Агент игрока: обеспечивает клиентский запуск боев и взаимодействие с серверами.
